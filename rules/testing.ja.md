# テスト要件

## 最小テストカバレッジ: 80%

テストタイプ (すべて必須):
1. **単体テスト (Unit Tests)** - 個々の関数、ユーティリティ、コンポーネント
2. **統合テスト (Integration Tests)** - APIエンドポイント、データベース操作
3. **E2Eテスト (E2E Tests)** - 重要なユーザーフロー (Playwright)

## テスト駆動開発 (Test-Driven Development)

必須ワークフロー:
1. 最初にテストを書く (RED)
2. テストを実行 - 失敗するはず
3. 最小限の実装を書く (GREEN)
4. テストを実行 - パスするはず
5. リファクタリング (IMPROVE)
6. カバレッジを検証 (80%+)

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使用する
2. テストの分離を確認する
3. モックが正しいことを確認する
4. テストではなく実装を修正する (テストが間違っている場合を除く)

## エージェントサポート

- **tdd-guide** - 新機能のために積極的に使用し、テスト・ビフォー・コードを強制する
- **e2e-runner** - Playwright E2Eテストスペシャリスト
