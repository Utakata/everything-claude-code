# コードマップの更新

コードベース構造を分析し、アーキテクチャドキュメントを更新する：

1. インポート、エクスポート、依存関係についてすべてのソースファイルをスキャンする
2. 以下のフォーマットでトークン効率の良いコードマップを生成する：
   - codemaps/architecture.md - 全体アーキテクチャ
   - codemaps/backend.md - バックエンド構造
   - codemaps/frontend.md - フロントエンド構造
   - codemaps/data.md - データモデルとスキーマ

3. 前のバージョンからの差分率を計算する
4. 変更が30%を超える場合、更新前にユーザーの承認を求める
5. 各コードマップに鮮度タイムスタンプを追加する
6. レポートを .reports/codemap-diff.txt に保存する

分析にはTypeScript/Node.jsを使用する。実装の詳細ではなく、高レベルの構造に焦点を当てる。
