# Refactor Clean

テスト検証を行いながら、デッドコードを安全に特定して削除する：

1. デッドコード分析ツールを実行:
   - knip: 未使用のエクスポートとファイルを見つける
   - depcheck: 未使用の依存関係を見つける
   - ts-prune: 未使用のTypeScriptエクスポートを見つける

2. .reports/dead-code-analysis.md に包括的なレポートを生成する

3. 発見事項を重大度で分類する:
   - SAFE (安全): テストファイル、未使用ユーティリティ
   - CAUTION (注意): APIルート、コンポーネント
   - DANGER (危険): 設定ファイル、メインエントリーポイント

4. 安全な削除のみを提案する

5. 各削除の前に:
   - 完全なテストスイートを実行
   - テスト合格を検証
   - 変更を適用
   - テストを再実行
   - テストが失敗した場合はロールバック

6. クリーンアップされた項目の概要を表示する

最初にテストを実行せずにコードを決して削除しないこと！
