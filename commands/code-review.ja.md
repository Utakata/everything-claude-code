# コードレビュー

未コミットの変更に対する包括的なセキュリティと品質のレビュー：

1. 変更されたファイルを取得: `git diff --name-only HEAD`

2. 各変更ファイルについて、以下をチェック:

**セキュリティ問題 (CRITICAL - 重大):**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力検証の欠如
- 安全でない依存関係
- パストラバーサルリスク

**コード品質 (HIGH - 高):**
- 50行を超える関数
- 800行を超えるファイル
- 4レベルを超えるネスト深度
- エラーハンドリングの欠如
- console.log ステートメント
- TODO/FIXME コメント
- パブリックAPIのJSDoc欠如

**ベストプラクティス (MEDIUM - 中):**
- 変更パターン (代わりに不変パターンを使用)
- コード/コメントでの絵文字使用
- 新しいコードに対するテスト欠如
- アクセシビリティ問題 (a11y)

3. 以下のレポートを生成:
   - 重大度: CRITICAL, HIGH, MEDIUM, LOW
   - ファイルの場所と行番号
   - 問題の説明
   - 推奨される修正

4. CRITICAL または HIGH の問題が見つかった場合、コミットをブロックする

セキュリティ脆弱性のあるコードは決して承認しないこと！
