# ECCのスポンサーになる

ECCは、Claude Code、Cursor、OpenCode、およびCodexアプリ/CLIにわたる、オープンソースのAIエージェントハーネスパフォーマンスシステムとして維持されている。

## スポンサーになる理由

スポンサーシップは以下を直接的に資金援助する。

- より迅速なバグ修正とリリースサイクル
- ハーネス間でのクロスプラットフォームの同等性作業
- コミュニティに無料で提供され続ける公開ドキュメント、スキル、および信頼性ツール

## スポンサーシップのティア

これらは実践的な出発点であり、パートナーシップの範囲に応じて調整可能である。

| ティア | 価格 | 最適な対象 | 含まれるもの |
|------|-------|----------|----------|
| パイロットパートナー (Pilot Partner) | $200/月 | 初めてのスポンサーエンゲージメント | 月次の指標更新、ロードマップのプレビュー、優先的なメンテナーからのフィードバック |
| グロースパートナー (Growth Partner) | $500/月 | ECCを積極的に採用しているチーム | パイロットの特典 + 月次のオフィスアワー同期 + ワークフロー統合のガイダンス |
| ストラテジックパートナー (Strategic Partner) | $1,000+/月 | プラットフォーム/エコシステムのパートナーシップ | グロースの特典 + 調整されたローンチサポート + メンテナーとのより深いコラボレーション |

## スポンサー向けレポート

毎月共有される指標には、以下のものが含まれる。

- npmダウンロード数 (`ecc-universal`, `ecc-agentshield`)
- リポジトリの採用状況 (スター、フォーク、コントリビューター)
- GitHub Appのインストール傾向
- リリースペースと信頼性のマイルストーン

正確なコマンドスニペットと再現可能な取得プロセスについては、[`docs/business/metrics-and-sponsorship.md`](docs/business/metrics-and-sponsorship.md)を参照すること。

## 期待事項と範囲

- スポンサーシップはメンテナンスと開発の加速をサポートするものであり、プロジェクトの所有権を譲渡するものではない。
- 機能リクエストは、スポンサーティア、エコシステムへの影響、およびメンテナンスリスクに基づいて優先順位付けされる。
- セキュリティと信頼性の修正は、全く新しい機能よりも優先される。
