# ユーザーレベルCLAUDE.mdの例

これはユーザーレベルの CLAUDE.md ファイルの例である。`~/.claude/CLAUDE.md` に配置する。

ユーザーレベルの設定はすべてのプロジェクトにグローバルに適用される。以下の用途に使用する：
- 個人的なコーディングの好み
- 常に強制したい普遍的なルール
- モジュール化されたルールへのリンク

---

## コア哲学

あなたは Claude Code である。私は複雑なタスクのために専門のエージェントとスキルを使用する。

**主要な原則:**
1. **エージェントファースト**: 複雑な作業は専門のエージェントに委任する
2. **並列実行**: 可能な場合は複数のエージェントでタスクツールを使用する
3. **実行前に計画**: 複雑な操作には計画モードを使用する
4. **テスト駆動**: 実装の前にテストを書く
5. **セキュリティファースト**: セキュリティに関しては決して妥協しない

## 個人的な好み

### コーディングスタイル
- コード、コメント、ドキュメントに絵文字を使用しない
- 不変性を好む - オブジェクトや配列を決して変更しない
- 少数の大きなファイルより、多数の小さなファイル
- 1ファイルあたり通常200-400行、最大800行

### Git
- Conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- コミット前に常にローカルでテストする
- 小さく、焦点を絞ったコミット

### テスト
- TDD: テストを最初に書く
- 最低80%のカバレッジ
- 重要なフローのための単体 + 統合 + E2E

以下の場合に成功とする：

- セキュリティ脆弱性がない
- コードが読みやすく保守可能である
- ユーザー要件が満たされている
